# Glossary

Definitions of terms used throughout the Obsidian Smart Composer codebase and documentation.

---

## A

### ADR (Architecture Decision Record)
A document that captures an important architectural decision made along with its context and consequences. See [ADR.md](../../ADR.md).

### Anthropic
AI company that created Claude. One of the supported LLM providers. SDK version 0.39.0.

### API Key
Secret credential used to authenticate with LLM provider services. Stored encrypted in plugin settings.

---

## B

### Block (Mentionable)
A specific selection of text within a file that can be referenced in chat. Includes file, start line, and end line.

---

## C

### Chat Message
A single message in a conversation. Can be `user`, `assistant`, or `tool` role.

### Claude Code
Anthropic's Claude configured for coding tasks. Supported as a provider type.

### Completion
The response generated by an LLM to a prompt. May be streamed or returned as a single response.

### Content Part
A component of a message that can be text, image, or tool result. Used in multimodal conversations.

### Context Provider
A React component that provides data to child components via React Context API. The plugin uses nested providers for settings, database, RAG, etc.

---

## D

### Drizzle ORM
TypeScript ORM used for database operations. Version 0.39.0. Chosen for type safety and browser compatibility.

### DatabaseManager
Singleton class that manages the PGlite database connection and provides access to the Drizzle instance.

---

## E

### Embedding
A vector representation of text, typically 1536 dimensions for OpenAI's text-embedding-3-small. Used for semantic similarity search.

### Embedding Model
An LLM specifically designed to create embeddings. Examples: `text-embedding-3-small`, `text-embedding-3-large`.

### esbuild
Fast JavaScript/TypeScript bundler used for building the plugin. Version 0.17.3.

---

## F

### Factory Pattern
Design pattern used for creating LLM provider instances. See `manager.ts`.

### Frontmatter
YAML metadata at the beginning of a markdown file. Used by Obsidian for file properties.

### Fuzzysort
Library for fuzzy string matching. Used in file search and mentionable autocomplete. Version 3.1.0.

---

## G

### Gemini
Google's LLM family. Supported via @google/generative-ai SDK.

### Groq
Fast inference provider for open-source models. Supported via groq-sdk.

---

## H

### HNSW (Hierarchical Navigable Small World)
Vector index algorithm used by pgvector for fast approximate nearest neighbor search. Configured with m=16, ef_construction=64.

### Hook
A React function that starts with `use` and provides stateful logic. Examples: `useSettings`, `useDatabase`.

---

## I

### In-Memory
Data stored in RAM rather than persistent storage. The plugin uses persistent PGlite storage, not in-memory.

---

## J

### JSON Database
The plugin stores chats and templates as JSON files in `.smtcmp_json_db/` folder.

### js-tiktoken
JavaScript port of OpenAI's tiktoken tokenizer. Used for counting tokens before API calls. Version 1.0.15.

---

## L

### LangChain
Framework for building LLM applications. Used for text splitting (`RecursiveCharacterTextSplitter`). Version 0.3.2.

### Lexical
Meta's extensible text editor framework. Used for the chat input component. Version 0.17.1.

### LLM (Large Language Model)
AI model trained on text that can generate human-like responses. Examples: GPT-4, Claude, Llama.

### LM Studio
Local LLM hosting application. Supported as a provider type.

---

## M

### MCP (Model Context Protocol)
Anthropic's protocol for LLM tool use. Enables structured tool calls and responses. SDK version 1.9.0.

### Mentionable
Something that can be referenced in a chat message using @-syntax. Types: file, folder, vault, current-file, block, url, image.

### Migration
Code that transforms data from one schema version to another. Used for both settings and database schemas.

### Model
A specific LLM configuration (e.g., gpt-4o, claude-sonnet-4-20250514). Each provider can have multiple models.

---

## N

### Node.js
JavaScript runtime used for development. Required v18+.

---

## O

### Obsidian
Note-taking application built on Electron. The target platform for this plugin.

### Ollama
Local LLM runner. Supported as a provider type. Requires CORS configuration.

### OpenAI
Company that created GPT models. Primary LLM provider. SDK version 4.91.1.

### OpenRouter
LLM API aggregator that provides access to multiple models. Supported as a provider type.

---

## P

### PGlite
WebAssembly build of PostgreSQL. The plugin's vector database. Version 0.2.12.

### pgvector
PostgreSQL extension for vector similarity search. Compiled into PGlite. Supports HNSW indexing.

### Provider
An LLM service that the plugin can connect to. Examples: OpenAI, Anthropic, Ollama.

### ProviderClient
Interface implemented by all LLM providers. Defines methods like `streamResponse`, `generateResponse`, `getEmbedding`.

---

## Q

### Query Vector
The embedding of the user's search query, used to find similar documents in the vector database.

---

## R

### RAG (Retrieval-Augmented Generation)
Technique that retrieves relevant documents and includes them in LLM context. Core feature of the plugin.

### RAGEngine
Class that coordinates indexing and retrieval of vault content.

### React
JavaScript UI library. Version 18.3.1. Used for all plugin UI components.

### React Query
Data fetching and caching library. Version 5.56.2. Used for async state management.

### Reindex
Process of re-creating all embeddings for vault files. Triggered manually or when embedding model changes.

---

## S

### Schema
The structure definition for data. Used for both database tables (Drizzle) and settings (Zod).

### SerializedEditorState
Lexical's format for persisting editor content. Stored in chat messages.

### Settings
Plugin configuration stored in Obsidian's `data.json`. Versioned with migrations.

### Similarity Search
Finding documents whose embeddings are close to a query embedding. Uses cosine distance.

### Streaming
Receiving LLM responses incrementally as they're generated. Provides better UX than waiting for full response.

### stdio Transport
MCP's communication method using standard input/output. Used for local tool servers.

---

## T

### TFile
Obsidian's type representing a file in the vault.

### TFolder
Obsidian's type representing a folder in the vault.

### Template
Reusable chat prompt configuration. Stored as JSON in `.smtcmp_json_db/templates/`.

### Tiktoken
OpenAI's tokenizer library. JavaScript port used for token counting.

### Tool
An MCP capability that can be invoked by the LLM. Examples: file_read, web_search.

### Tool Call
An LLM request to execute a specific tool with arguments.

### TypeScript
Typed superset of JavaScript. Version 5.9.3. Used for all source code.

---

## U

### UUID
Universally Unique Identifier. Used for chat and message IDs. Generated via uuid package.

---

## V

### Vault
Obsidian's term for a collection of notes in a folder.

### Vector
A numerical array representing text semantically. Typically 1536 dimensions.

### Vector Database
Database optimized for storing and searching vectors. The plugin uses PGlite + pgvector.

### VectorManager
Class that handles vector storage and similarity search operations.

### vscode-diff
Library for computing text differences. Used for applying code changes. Version 2.1.1.

---

## W

### WASM (WebAssembly)
Binary instruction format for web browsers. PGlite runs PostgreSQL compiled to WASM.

### Worker
Web Worker or separate process that runs code without blocking the main thread.

---

## Z

### Zod
TypeScript-first schema validation library. Version 3.23.8. Used for settings validation and API schemas.
